Section 01

핵심 결정 사항

공무원 개개인이 아무것도 설치하지 않고도 바로 쓸 수 있도록 Node.js 이식형(Portable)을 ZIP 안에 번들한다.

런타임 결정
Node.js 이식형 번들
nodejs.org 공식 Windows ZIP(node.exe 포함)을 배포 패키지 안에 포함. 시스템 Node.js 설치 불필요.
배포 방식 결정
ZIP 압축 해제 + 실행.bat
배포자가 준비한 ZIP을 압축 해제하면 모든 것이 갖춰진다. 이후 실행.bat 더블클릭 하나로 시작.
출력물 결정
.md + YAML frontmatter
Markdown 단독 출력. 페이지·슬라이드 마커 포함. blocks.json 등 부가 파일 없이 단순명료하게.
⚠ 개정 이력 v1 Python 중심 → v2 kordoc 채택·혼합 런타임 → v3 Node.js 단독·개인 PC → v4 (현재): 이식형 Node.js 번들 → 설치 자체를 없앰

Section 04

실행.bat — 전체 코드

공무원이 더블클릭하는 파일 전체 코드다. 서버 시작과 브라우저 오픈이 자동으로 완료된다.

@echo off
chcp 65001 > nul                 :: UTF-8 인코딩 설정
cd /d "%~dp0"                    :: bat 파일 위치를 기준 폴더로 설정

echo.
echo  ■ 제안서 MD 변환 도구
echo    브라우저가 자동으로 열립니다.
echo    이 창을 닫으면 도구가 종료됩니다.
echo.

:: 번들된 Node.js 이식형 사용 (시스템 설치 불필요)
set NODE=.\runtime\node.exe

:: 번들 파일 존재 확인
if not exist "%NODE%" (
    echo  [오류] runtime\node.exe 를 찾을 수 없습니다.
    echo         ZIP 파일을 다시 압축 해제해 주세요.
    pause
    exit /b 1
)

:: 포트 충돌 확인 (이미 실행 중인지)
netstat -an | find ":3000" > nul 2>&1
if %errorlevel% == 0 (
    echo  [안내] 이미 실행 중입니다. 브라우저에서 확인하세요.
    start http://localhost:3000
    pause
    exit /b 0
)

:: 서버 시작 (백그라운드)
start /b "%NODE%" app.js

:: 서버 준비 대기 후 브라우저 열기
timeout /t 2 /nobreak > nul
start http://localhost:3000

echo  실행 중... 이 창을 닫으면 도구가 종료됩니다.
pause > nul
관리자 권한 불필요 runtime\node.exe는 시스템 PATH나 레지스트리를 건드리지 않는다. 일반 사용자 계정으로도 실행된다.

Section 09

배포자(개발자)가 ZIP을 준비하는 방법

공무원이 아무것도 안 해도 되려면, 배포자가 인터넷 PC에서 미리 모든 것을 ZIP에 담아야 한다. 이 작업은 한 번만 한다.

1
Node.js 이식형 다운로드
nodejs.org에서
Windows Binary ZIP
다운로드
2
앱 패키지 설치
앱 폴더에서
npm install
node_modules/ 생성
3
폴더 조합
Node ZIP 압축 해제
→ runtime/ 폴더로
소스와 합치기
4
ZIP 압축 + 반입
전체 폴더 압축
→ 내부망 반입
→ 공유 폴더 배포
:: ── 배포자 준비 명령어 (인터넷 PC에서 실행) ──────────────

:: 1. Node.js 이식형 다운로드 (nodejs.org 공식)
curl -o node18.zip https://nodejs.org/dist/v18.20.0/node-v18.20.0-win-x64.zip
:: 압축 해제 후 폴더명을 runtime/ 으로 변경

:: 2. 앱 의존성 설치
npm install          :: node_modules/ 생성

:: 3. ZIP 압축 (PowerShell)
Compress-Archive -Path .\제안서MD변환도구 -DestinationPath 제안서MD변환도구_v1.zip
⚠ 망분리 반입 시 보안 검토 항목 node-v18.x.x-win-x64.zip (nodejs.org 공식 배포본, SHA256 해시 검증 가능), node_modules/ 하위 패키지 목록 제출 (kordoc·pdfjs-dist·officeparser·express·multer). kordoc은 MIT 오픈소스, ZIP bomb·XXE 방어 내장 확인됨.

kordoc MCP 서버 연동 → Claude Desktop에서 직접 "이 제안서 평가항목별 정리해줘" 실행. 평가항목별 근거 추출, 업체 간 비교표 자동 생성.
Section 11

리스크 및 대응

리스크수준대응 방안
kordoc 신생 프로젝트 (v0.1 출시 7주, ⭐243)중1단계 실제 제안서 10건 이상 집중 검증. 실패율 높으면 converter/hwpx.js만 교체.
kordoc PDF 표 품질 미달 가능성중샘플 테스트 후 미달이면 pdf.js를 pymupdf4llm Python subprocess 호출로 교체. Node.js only 원칙을 PDF에 한해 완화.
ZIP 크기 과다 (pdfjs-dist 비중)낮음불필요한 wasm·locale 파일 제거로 경량화. 목표: 압축 후 65MB 이하.
망분리 반입 보안 승인 지연높음보안 담당자와 사전 협의 및 파일 목록 제출 선행. 지연 시 개발자 PC에서 CLI 변환 후 MD 파일만 공유하는 임시 운영.
Windows 7/8 환경 호환낮음Node.js 18은 Windows 10 이상 지원. Windows 7/8이면 Node.js 14 LTS로 다운그레이드.
